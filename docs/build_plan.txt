STEP 2 — DATA MODELS (cogmem/models/)
================================================================================
Status: [ ] Not started

  common.py
    - MemoryType enum (episodic, semantic, spatial, pattern, emotional, prospective)
    - Tier enum (repo, workspace, global)
    - Phase enum (vivid, clear, fuzzy, fading, stub)
    - EmotionType enum (pain, danger, trust, pride, frustration, relief, curiosity)
    - DangerLevel enum (low, medium, high, critical)
    - TaskType enum (bugfix, feature, refactor, understand, plan, explore, cross_repo)
    - RetrievalBias enum (boost, demote, warn)
    - GistScope enum (platform, codebase, module, component)
    - PatternCategory enum (bug, design, anti, smell, cross_repo)
    - Base model with to_markdown() and from_markdown() support

  episode.py
    - Episode model
    - Fields: id, date, when, tier, trigger, story, learned, emotion,
      intensity (0-1), code_touched [], repos_involved [], people_involved [],
      related_episodes [], related_patterns [], triggered_prospective [],
      source (lived|git-inferred|auto-generated), source_confidence,
      strength, access_count, last_accessed, phase
    - Three compression levels: full (~200 tok), summary (~50 tok), stub (~15 tok)
    - Phase lifecycle logic: vivid (<7d), clear (7-30d), fuzzy (30-90d),
      fading (90-180d), stub (>180d)

  gist.py
    - Gist model
    - Fields: id, scope, target, tier, what_it_does, why_it_exists,
      how_it_works, key_relationships, judgment (fragile|solid|improving|unknown),
      confidence, formed_from [], contradictions [], last_updated, last_verified
    - Four scope levels:
        Level 0 — Platform identity (workspace, ~300 tokens)
        Level 1 — Codebase identity (~200 tokens)
        Level 2 — Module understanding (~100 tokens)
        Level 3 — Component understanding (~50 tokens)

  pattern.py
    - Pattern model
    - Fields: id, name, category, tier, signature, consequence, response,
      seen_in [], frequency, last_seen, trigger_cues [], strength,
      danger_level, transferable

  emotion.py
    - EmotionTag model
    - Fields: target, emotion, intensity, reason, source_episodes [],
      last_reinforced, retrieval_bias, zoom_override (full|summary|null)

  spatial.py
    - RepoSpatial model
      Fields: surface [], middle [], deep [], landmarks [], neighborhoods []
      Each entry: {path, description, feel}
    - WorkspaceSpatial model
      Fields: services [], data_flow [], shared_contracts [],
      deployment, danger_zones []

  prospective.py
    - Prospective model
    - Fields: id, intention, trigger, trigger_type, created_from,
      priority, completed

  entity.py
    - CodeEntity model
    - Fields: id, file_path, name, kind (function|class|method),
      signature (~10 tokens), summary (~50 tokens)
    - REPO-LEVEL ONLY (too granular for workspace/global)

STEP 5 — UTILITIES (cogmem/utils/)
================================================================================
Status: [ ] Not started

  repo_detect.py
    - find_repo_root(path) -> path or None
      Walk up directories looking for .git/
    - find_workspace(path) -> path or None
      Walk up looking for .cogmem/ or detect multiple .git/ siblings
    - detect_repos_in_dir(path) -> [repo_paths]
      Find all immediate child directories containing .git/

  cues.py
    - extract_cues(query: str) -> CueSet
    - CueSet: keywords [], entities [], file_paths [], emotions [],
      repo_references [], task_type_hints []
    - Regex patterns for:
      File paths, function names, class names
      Emotion words (pain, bug, broken, dangerous, etc.)
      Task type words (fix, add, refactor, understand, etc.)
      Repo name references

  scoring.py
    - compute_score(memory, cues, tier_weight, priming_context) -> float
    - Formula: keyword_match * convergence_multiplier * emotional_weight
               * recency_weight * context_congruence * priming_bonus
               * tier_adjustment
    - Tier weights: current_repo=1.0, workspace=0.9, global=0.8, other_repo=0.7
    - Recency weight: exponential decay based on last_accessed

  tokens.py
    - estimate_tokens(text: str) -> int
      Rough: len(text) / 4
    - truncate_to_budget(text: str, budget: int) -> str
    - BUDGET_TABLE: dict mapping TaskType -> {episodes%, gist%, patterns%,
      dangers%, entities%, spatial%, prospective%}
      (values from spec Section 20, Stage 6)

STEP 6 — ENGINE: RECALL (cogmem/engine/recall.py)
================================================================================
Status: [ ] Not started

  def recall(query, task_type=None, files=None, budget=3500, cwd=None):

    Stage 1 — Cue extraction (<10ms)
      cues = extract_cues(query)
      if files: add file-derived cues

    Stage 2 — Determine search scope
      current_repo = find_repo_root(cwd)
      workspace = find_workspace(cwd)
      scopes = [("repo", current_repo, 1.0)]
      if workspace:
          scopes.append(("workspace", workspace, 0.9))
          for repo in workspace.repos if cues reference them:
              scopes.append(("repo", repo, 0.7))
      scopes.append(("global", global_path, 0.8))

    Stage 3 — Multi-tier search (<100ms)
      For each scope: query keyword index, load matching memories

    Stage 4 — Convergence scoring
      Score each match using compute_score()

    Stage 5 — Task type detection
      If not provided: detect from cues (bugfix/feature/refactor/etc.)

    Stage 6 — Budget allocation
      Distribute budget across memory types per BUDGET_TABLE[task_type]
      Select top memories within each type's budget

    Stage 7 — Assembly + metamemory
      Format output with sections: Understanding, Past Experiences,
      Danger Warnings, Patterns, Reminders, Cross-repo Context
      Add metamemory notes (confidence, cold spots)

  Returns: formatted string (~3500 tokens)

STEP 8 — ENGINE: CONSOLIDATION (cogmem/engine/consolidate.py)
================================================================================
Status: [ ] Not started

  def consolidate(scope="full", cwd=None):

    if scope in ("full", "repo"):
      consolidate_repo(repo):
        Phase 1 — Episode compression
          Move episodes to appropriate phase based on age
          Compress narrative at each transition
        Phase 2 — Pattern extraction
          Find recurring themes across episodes
          Create/strengthen pattern memories
        Phase 3 — Gist formation
          Update module/component gists from episodes
        Phase 4 — Emotional recalibration
          Adjust intensity based on recent reinforcement
        Phase 5 — Pruning
          Delete stub episodes that have been consolidated
          Remove orphaned entities

    if scope in ("full", "workspace"):
      consolidate_workspace(workspace):
        Phase W1 — Cross-repo pattern detection
          Prepare context from all repos' recent episodes
          Write prompt to .cogmem/.pending/consolidate_prompt.md
          Return instructions for subagent
        Phase W2 — Workspace gist update
          Check if repo gists changed significantly
        Phase W3 — Cross-repo emotional recalibration
          Adjust workspace danger tags

    if scope in ("full", "global"):
      consolidate_global():
        Find repo patterns with transferable=true
        Copy to ~/.cognitive-memory/patterns/ if not already there

  def consolidate_apply(result_file):
    - Parse subagent result
    - Apply changes to workspace memory

  SUBAGENT PATTERN:
    cogmem consolidate writes prompt to .pending/
    Returns message: "Run consolidation subagent to process"
    Subagent skill reads prompt, processes, writes result
    cogmem consolidate --apply <result> applies changes

STEP 9 — ENGINE: DECAY (cogmem/engine/decay.py)
================================================================================
Status: [ ] Not started

  DECAY_RATES = {
    "episode":     0.05,    # half-life ~14 days
    "gist":        0.005,   # half-life ~140 days
    "pattern":     0.01,    # half-life ~70 days
    "emotion":     0.002,   # half-life ~350 days
    "emotion_pain": 0.001,  # half-life ~700 days
    "spatial":     0.003,   # half-life ~230 days
    "prospective": 0.02,    # half-life ~35 days
  }

  def run_decay(cwd=None):
    - Load all memories from repo tier
    - Apply strength = strength * exp(-rate * days_since_access)
    - Workspace memories: rate *= 0.7
    - Global patterns: no decay
    - Trigger phase transitions for episodes
    - Compress episodes at phase transitions
    - Save updated memories

  def compute_new_strength(current, rate, days_elapsed):
    return current * math.exp(-rate * days_elapsed)

  def transition_episode_phase(episode):
    age = (today - episode.date).days
    new_phase = vivid if age < 7, clear if < 30, fuzzy if < 90,
                fading if < 180, stub otherwise
    if new_phase != episode.phase:
      compress episode narrative
      update episode.phase

STEP 10 — ENGINE: BOOTSTRAP (cogmem/engine/bootstrap.py)
================================================================================
Status: [ ] Not started

  def bootstrap_repo(repo_path, months=6):

    Step 1: File tree -> spatial map
      Walk directory tree, classify into surface/middle/deep
      Identify landmarks (README, main entry, config)
      Save to .memory/spatial.md

    Step 2: README -> codebase gist
      Read README.md if exists
      Write prompt to .memory/.pending/bootstrap_gist_prompt.md
      (Subagent generates gist from README)
      Fallback: create minimal gist from file tree

    Step 3: Source per module -> module gists
      Identify top-level source directories
      Write prompts for each module
      (Subagent generates module gists)

    Step 4: Git log -> synthetic episodes
      git log --since={months}m --format="%H|%s|%an|%ad|%D" --name-only
      Group by theme (branch, related files)
      Create Episode objects (source="git-inferred")

    Step 5: Commit patterns -> emotional tags
      Reverts, hotfixes -> pain
      Long-lived files with many fixes -> danger
      Stable modules -> trust

    Step 6: Code analysis -> entity summaries
      Parse source files for function/class signatures
      Create CodeEntity objects (signature only, no LLM needed)

    Step 7: Episode clustering -> initial patterns
      Group episodes by files/themes
      Detect recurring bug patterns

    Step 8: Build keyword index
      Scan all created memory files
      Build keyword_index.json

    Step 9: Scan @memory: annotations
      Grep source for @memory: comments

    Step 10: Import global patterns
      Copy from ~/.cognitive-memory/patterns/ if exists

  def bootstrap_workspace(workspace_path, months=6):

    Step 1: Detect all repos (find .git/ directories)
    Step 2: Bootstrap each repo independently
    Step 3: Analyze cross-repo relationships
      Scan imports, shared deps, docker-compose, CI configs
      Write prompt for subagent
    Step 4: Generate workspace memory
      Create .cogmem/gist/_platform.md
      Create .cogmem/spatial.md
      Create .cogmem/meta.json
    Step 5: Find cross-repo episodes from git
      Correlate commits across repos by time/branch/ticket
    Step 6: Detect cross-repo danger zones
      Shared contracts, API boundaries
    Step 7: Build workspace keyword index
